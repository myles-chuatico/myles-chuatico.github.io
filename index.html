eLinq Utilization Reporting Pipeline
The financial support department had no visibility into eLinq system adoption — AVPs were noticing declining call volumes but had no data to act on. I built the entire pipeline from scratch to fix it.
Problem
Log files scattered across all agent machines with no centralized collection
Solution
Batch script for automated daily log collection → Power Query cleaning pipeline → SAS data matching → weekly leadership reports
Outcome
Measurably improved eLinq adoption; gave team leaders actionable data to coach low-utilization agents
Stack

Solution
Power BI dashboards with direct FinCoorP API integration — daily refresh, interactive filtering, shareable via Teams/SharePoint

A branching yes/no decision tree that guides agents to the correct resolution pathway based on account status, arrears markers, and customer circumstances. Integrated calculators handle plan-specific financial formulas at terminal nodes.
