Implementation Detail
Constraint Analysis
Solution Design Under Restrictions
Requirements gathering revealed that agent machines were restricted corporate endpoints with no third-party tools and no elevated privileges. Assessed available options and identified Windows Task Scheduler as the only viable mechanism. Designed the collection script around these constraints: runs before end of shift, handles file naming (agent ID + date) for attribution, prevents overwrites.
:: Daily eLinq log push
@echo off
set DATE=%date:~-4%-%date:~4,2%
xcopy "%LOCALAPPDATA%\eLinq\*.log"
    "\\server\shared\elinq-logs\"
    /Y /Q
Data Transformation
ETL Specification & Implementation
Defined the transformation requirements: raw logs needed parsing, standardization, deduplication, and agent attribution. Implemented in Power Query M, connecting to the shared folder with automated daily refresh.
// Power Query M steps
1. Folder.Files(shared_path)
2. Combine & expand all logs
3. Parse timestamp column
4. Remove null/error rows
5. Add agent_id from filename
6. Pivot by date + agent
Quality Assurance
Ongoing Validation & Stakeholder Support
Established daily validation checks: file count vs agent roster, timestamp integrity, refresh status, SAS match deltas. Diagnosed and resolved individual agent issues (network errors, permissions). Authored and distributed a troubleshooting guide department-wide to reduce support requests.
